# 타자 아레나 (Typing Arena)

옛 한국 타자 게임 **'베네치아'** 를 모티브로 한 실시간 타자 대결 게임.
하늘에서 떨어지는 한국어(또는 영어) 단어를 입력해 파괴하고, 막지 못해 바닥(물)에 닿으면
도시가 한 칸씩 **수몰**된다. 라이프 9에서 시작, 모두 잠기면 게임 오버.
(폴더/URL 은 `Venice/` 로 유지 — 표시 이름만 '타자 아레나'.)

- 🎮 **플레이:** https://garamisp.github.io/webbuilds/Venice/
- 순수 웹(HTML5 Canvas + Vanilla JS). 빌드 단계 없음. GitHub Pages 정적 호스팅.
- 멀티는 Railway 의 **중개 서버**(WebSocket relay)를 거친다 → `server/` 참고.

## 플레이 방법
- 떨어지는 단어를 **그대로 입력**하면 그 단어가 파괴된다(폭발 연출).
- 같은 단어가 여러 개 떠 있어도 **가장 아래쪽 한 개**만 파괴된다.
- 입력 중인 글자와 일치하는 단어는 노란색으로 강조된다.
- `Enter` 는 유지(오타 수정용), `Esc` 는 입력 비우기.
- 단어가 물에 닿으면 **라이프 −1**, 물이 차오른다. 라이프 0 → 수몰 → 게임 오버.
- 대결에서는 **내가 자연 생성한 단어를 깰 때** 그 단어가 상대 모두에게 발사된다.
  (받은 공격·발사 스킬 단어를 깨는 건 방어 전용 → 무한 핑퐁 없음.)

### ⚡ 스페셜 스킬 — 단어 발사
- **30초마다 1충전**(최대 3). 충전이 있으면 상단 ⚡ 버튼이 켜진다.
- `Tab` 또는 ⚡ 버튼 → 단어 입력창. **최대 10자, 공백 불가.** 말 안 되는 단어도 OK(예: `우갸갸갹`).
- 발사하면 그 단어가 **상대 모두에게** 떨어지고, **나에게도** 떨어진다(보라색).
  → 내가 칠 수 있는 단어를 골라 상대를 흔드는 심리전.

## 모드
| 모드 | 설명 |
|---|---|
| **공개 (대결 매칭)** | 혼자 시작하더라도 같은 공개방에 다른 사람이 들어오면 **레벨이 초기화되고 자동 대결**. 단어를 맞추면 상대 모두에게 발사. 옆에 상대 화면이 작게 표시되어 내 공격이 꽂히는 걸 본다. **끝없는 아레나** — 수몰된 사람은 파괴 연출과 함께 게임 오버(다시 도전 가능), 살아남은 사람들은 "☠ XX 게임오버!" 알림과 **라이프 +1**을 받고 계속 싸운다. 계속 새 도전자가 들어온다. |
| **비공개 (혼자 레벨업)** | 매칭 없이 혼자. 시간이 갈수록 레벨·속도가 계속 상승. |

> 서버에 연결돼 있어도 상대가 없으면 그냥 혼자(속도 상승) 진행하다가,
> 누군가 들어오는 순간 대결이 시작된다. 서버가 없거나 연결 실패해도 혼자 모드로 정상 동작.

## 한국어 처리
- 모든 파일 UTF-8, `<meta charset="utf-8">`. 캔버스 폰트는 한글 우선 스택
  (Pretendard → Malgun Gothic → Apple SD Gothic Neo → Noto Sans KR).
- 입력은 숨김이 아닌 실제 `<input>` + `input`/`compositionend` 이벤트로 **OS 한글 IME 조합**을 그대로 사용.
- 네트워크 메시지(공격 단어·채팅·미니뷰)는 JSON(UTF-8)으로 왕복 — 서버 통합 테스트로 한글 보존 확인됨.

## 구조
```
Venice/
  index.html        # 화면/레이아웃
  css/style.css      # 스타일 (한글 폰트 스택)
  js/words.js        # 단어 데이터(한/영, 난이도 티어)
  js/effects.js      # 파티클/폭발/물보라 연출
  js/game.js         # 게임 엔진 (솔로+대결 공용)
  js/net.js          # 중개 서버 WebSocket 클라이언트 (솔로 폴백)
  js/main.js         # UI/입력(IME)/네트워크 연동/상대 미니뷰/채팅
  server/            # Railway 중개 서버 (Node + ws)
```

## 멀티 켜기 (1단계)
1. `server/` 를 Railway 에 배포 (→ `server/README.md`).
2. 발급된 도메인을 `js/net.js` 상단 `RAILWAY_URL` 에 `wss://...` 로 넣는다.
3. 끝. (테스트는 `?server=wss://...` 쿼리로 즉시 덮어쓰기 가능.)

## 로컬에서 돌려보기
```bash
# 1) 중개 서버
cd Venice/server && npm install && npm start      # ws://localhost:8787

# 2) 클라이언트 (정적 서버 아무거나)
cd Venice && python -m http.server 5599
#  → http://localhost:5599  (localhost 면 자동으로 localhost 서버에 붙음)
```
브라우저 탭 두 개로 열면(한쪽은 공개 시작) 대결을 확인할 수 있다.
